## 1. GitHub Skill
**Purpose**: Interact with GitHub repositories via CLI

**Key Commands**:
- `gh pr checks <number> --repo owner/repo` - Check CI status
- `gh pr list --repo owner/repo` - List open PRs
- `gh run list --repo owner/repo` - List workflow runs
- `gh api repos/owner/repo/pulls/55` - Custom API queries

**Setup**: Install `gh` CLI, run `gh auth login`

**Use Cases**: CI/CD monitoring, PR management, repository automation

---

## 2. Notion Skill
**Purpose**: Create, read, update Notion pages and databases

**Key Commands** (curl-based API):
- `POST /v1/search` - Search pages/databases
- `POST /v1/pages` - Create pages
- `POST /v1/data_sources/{id}/query` - Query databases
- `PATCH /v1/blocks/{page_id}/children` - Add content blocks

**Setup**: Create integration at notion.so/my-integrations, get API key (ntn_...)

**Use Cases**: Task management, note taking, data logging, content creation

---

## 3. Obsidian Skill
**Purpose**: Manage Obsidian vaults (plain Markdown notes)

**Key Commands**:
- `obsidian-cli search "query"` - Search note names
- `obsidian-cli search-content "query"` - Search inside notes
- `obsidian-cli create "Folder/Note" --content "..."` - Create notes
- `obsidian-cli move "old" "new"` - Move/rename with link updates

**Setup**: Install `obsidian-cli` via `brew install yakitrak/yakitrak/obsidian-cli`

**Use Cases**: Daily notes automation, knowledge base search, note refactoring

---

## 4. Spotify Player Skill
**Purpose**: Control Spotify playback from terminal

**Key Commands**:
- `spogo search track "query"` - Search tracks
- `spogo play/pause/next/prev` - Playback control
- `spogo device list/set` - Manage devices
- `spogo status` - Current playback

**Setup**: Install `spogo` (preferred) or `spotify_player`, requires Spotify Premium

**Use Cases**: Quick playback control, music discovery, multi-device management

---

## 5. Weather Skill
**Purpose**: Get weather info without API keys

**Key Commands**:
- `curl -s "wttr.in/London?format=3"` - One-liner current weather
- `curl -s "wttr.in/London?T"` - Full forecast
- `curl -s "wttr.in/JFK"` - Airport codes work

**Setup**: No setup needed! Just needs `curl`

**Use Cases**: Quick weather checks, travel planning, automation/scripts

---

## 6. SAG (ElevenLabs TTS) Skill
**Purpose**: High-quality text-to-speech with AI voices

**Key Commands**:
- `sag "Hello there"` - Speak text
- `sag -v "VoiceName" "Hello"` - Use specific voice
- `sag -o output.mp3 "Text"` - Save to file
- `sag voices` - List available voices

**Setup**: Install `sag` via `brew install steipete/tap/sag`, set `ELEVENLABS_API_KEY`

**Use Cases**: Voice responses, content creation, accessibility, character voices

**Voice Tags**: `[whispers]`, `[shouts]`, `[excited]`, `[laughs]`, etc.

## 7. TMUX Skill
**Purpose**: Remote control tmux sessions for interactive CLIs

**Key Commands**:
- `tmux -S "$SOCKET" new -d -s <session>` - Create session
- `tmux -S "$SOCKET" send-keys -t <target> -l -- "text"` - Send text
- `tmux -S "$SOCKET" capture-pane -p -t <target>` - Capture output
- `tmux -S "$SOCKET" kill-session -t <session>` - Cleanup

**Setup**: Requires `tmux` binary on PATH

**Use Cases**: Python REPL sessions, TUI apps (Claude Code, Codex), parallel coding agents

## 8. Healthcheck Skill
**Purpose**: Host security hardening and risk assessment

**Key Commands**:
- `openclaw security audit` - Basic security audit
- `openclaw security audit --deep` - Comprehensive audit
- `openclaw security audit --fix` - Apply safe defaults
- `openclaw update status` - Check version/updates

**Setup**: No special setup, uses OpenClaw CLI

**Use Cases**: Initial security audit, applying safe defaults, periodic monitoring, risk assessment
